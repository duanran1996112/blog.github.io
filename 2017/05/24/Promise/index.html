Promise

发表于 2017-05-24 | 分类于 Es6
Promise，异步执行 js 的方法

前述

在 JavaScript 的世界中，所有代码都是单线程执行的

由于这个“缺陷”，导致 JavaScript 的所有网络操作，浏览器事件，都必须是异步执行

异步执行需要大量回调函数，导致页面结构混乱，形成很可怕的回调地狱

而 Promise 就是用来解决回调函数这种“缺陷”，采用链式调用的方法异步执行 JavaScript

概念

ES6 原生提供了 Promise 对象

所谓 Promise，就是一个对象，用来传递异步操作的消息。

Promise 对象有三个状态：Pending，Resolved，Rejected

Pending（Promise 对象的起始状态，进行中）
Resolved（Promise 对象的成功状态）
Rejected（Promise 对象的失败状态）

Promise 对象的状态只可能从 Pending -> Resolved 或者 Pending -> Rejected，而且状态只要改变后不会再变

Promise 对象

创建一个 Promise 对象

Promise 对象接受一个函数作为参数，函数的形参有两个参数，resolve，reject，是原生 Js 中自带的方法

resolve：改变 Promise 的状态，变成成功状态，由（Pending -> Resolved），并且可以发送成功状态时的数据
reject：改变 Promise 的状态，变成失败状态，由（Pending -> Rejected），并且可以发送失败状态时的数据

例如：我们想在 成功 时将 Promise 的状态变为成功，并且发送 success，失败时将 Promise 的状态变为失败，并且发送 fail

Promise 对象的 成功 与 失败

在 Promise 对象的方法中
我们调用 resolve 这个函数，就可以将 Promise 的状态变为成功
我们调用 resject 这个函数，就可以将 Promise 的状态变为失败

例如：以 图片预加预加载 举例：当图片加载成功时我们把这个 img 标签发送，失败时我们发送 error 字符串
