http、https协议的区别

前言

http 是超文本传输协议，被用于在 web浏览器 与 网站服务器 之间传递信息。http 以明文的形式传递信息，不提供任何方式加密，如果 web攻击者 截取到了 web浏览器 与 网站服务器 的传输报文，就可以读懂其中的信息。因此 http协议 不适合传输一些敏感信息，例如：卡号、密码、支付信息。

为了解决 http协议 这种缺陷，我们需要另一种协议 https，为了数据传输安全，https协议 在 http协议 的基础上加入了 ssl 协议，ssl 靠证书来验证服务器身份，并为 浏览器 和 服务器 之间通信加密。

http 与 https 的区别

1.https 需要 CA（内含申请者身份信息、公钥、私钥） 证书，一般免费的证书少，因此需要一定的费用
2.http 信息是明文传输，https 信息是通过 TLS（SSL）加密的
3.http 与 https 使用的是不同的连接方式，所用的端口也不一样，http：80，https：443
4.http 的连接是简单的、无状态的，https 的连接是可加密的、含有身份状态的，比 http 协议安全

http 建立连接（三次握手）

1.客户端发送 SYN（SYN = 1，seq = j）包到服务器，客户端进入 SYN_SEND 状态，等待服务器确认。
2.服务器收到 SYN 包，确认客户端的 SYN 包从而发送 ACK（ACK = 1，ack = j + 1）包，同时自己也发送一个 SYN（SYN = 1，seq = k）包，即 SYN + ACK 包，服务器进入 SYN_RECV 状态，等待客户端确认。
3.收到 SYN + ACK 包，确认服务端的 SYN + ACK 包从而发送 ACK（ACK = 1，ack = k + 1）包，此包发送完毕，客户端 与 服务器 进入 ESTABLISHED 状态，完成三次握手

问题：三次握手为什么二次不行？
解答：为了实现可靠传输，避免资源的浪费。
详解：假设 客户端 向 服务端 传送一个报文，在某一个网络节点长时间滞留了，以至于延误了很长时间到达 服务端。本来是一个早已失效的报文段，而 服务端 误以为是 客户端 又发出一次新的连接请求，于是向 客户端 发出报文端确认，同意连接。假定不使用三次握手 服务端 已经建立连接，而由于 客户端 没有发出新的请求所以不会理睬 服务端，所以这样 服务端 的资源就白白浪费了。

https 建立连接 （传输层进行 TLS（SSL）加密）

1.客户端 发起 https 请求
2.服务端 配置数字证书（自制、CA），证书中含有公钥、私钥
3.服务端 向 客户端 传送证书，含有公钥
4.客户端 解析 证书，对证书中信息通过公钥解密，并与本机存储的信息进行对比，如果存在异常弹出警告，如果没有问题生成一个随机值，通过公钥进行加密
5.客户端 向 服务端 传送证书加密后的随机值
6.服务端 通过私钥解密该随机值，然后把内容通过该随机值进行加密
7.服务端 向 客户端 传输加密后的信息
8.客户端 通过随机值解密 服务端 传递过来的信息

问题：对称加密、非对称加密、HASH算法的区别？
解答：
对称加密 -> 加密与解密使用一对相同的密钥。（AES、DES）
非对称加密 -> 加密与解密使用不同的密钥，一个公钥，一个私钥，公钥加密的只有私钥才能解密，私钥加密的只有公钥才能解密。（RSA、DSA）
HASH算法 加密 -> 一种不可逆的加密，假设在 客户端 使用 HASH算法 加密用户信息，则 服务端事 先存储的用户信息已经为 HASH算法 的加密信息。（MD5、SHA1、SHA256）

http 断开连接（四次挥手）

1.客户端发送 FIN/ACK（FIN = 1，ACK = 1，seq = j）包到服务器，客户端进入 FIN_WAIT_1 状态。
2.服务端收到 FIN 包，确认客户端的 FIN 包从而发送 ACK（ACK = 1，ack = j + 1）包，服务器进入 CLOSE_WAIT 状态（半关闭状态），客户端进入 FIN_WAIT_2 状态。
3.服务端发送 FIN/ACK（FIN = 1，ACK = 1，seq = k）包到客户端，服务器进入 LAST-ACK 状态。
4.客户端收到 FIN 包，确认服务端的 FIN 包从而发送 ACK（ACK = 1，ack = k + 1）包，客户端进入 TIME_WAIT 状态，服务端进入 CLOSED 状态。

问题：四次挥手为什么两次不行？
解答：为了确保数据能够完成传输。
详解：未必在 客户端 要断开的时候，服务端 所有数据都传递完成，所以 服务端 要通过半关闭状态将剩余数据传递完成。

tcp 与 udp 协议的区别

  1. tcp 是可靠连接。udp 是不可靠连接，尽最大努力保证交付。
  2. tcp 是面向连接的。udp 不需要建立连接，发送数据前不需要建立连接。
  3. tcp 是一对一的连接。udp 是一对多的连接，udp 适用于建立视频会议。
  4. tcp 首部开销 20 个字节。udp 首部开销 8 个字节，比较小。
  5. tcp 面向字节，udp 面向报文。

http 1.x 与 2.0 协议的区别

  1. http2 采用二进制格式而非文本格式进行传输，二进制协议解析起来更高效。
  2. http2 报文头部进行压缩，降低了开销，采用 HPACK 首部压缩算法。
  3. http2 让服务器可以将响应主动推送到客户端缓存中，将一些外部引入的 js、css、img 文件在建立时推送到客户端的缓存。
  4. http2 是多路复用的，一个连接可以实现并行，同时处理多个请求和响应。
