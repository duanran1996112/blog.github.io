React 中知识点总结

react 与 vue 的区别

  1. 设计思想不同。react 是函数式的设计思想，把组件设计成纯组件，状态和逻辑通过参数传入，在 setState 之后会重新走渲染流程（生命周期）。vue 是响应式的设计思想，通过 watcher 来监听数据的变化，当发生变化时，响应式的更新对应的虚拟 dom。


  2. 编写方式不同。react 是 jsx 语法（all in js），通过 js 来操作一切。vue 是模板语法，把 html、css、js 写在一个文件中，同时含有自己的 html 模板引擎。

  3. 组件写法不同。react 是类（class）式写法，api 较少。vue 是声明式写法，通过传入各种 options（Vue.extend({…})），api 较多。
  4. 组件状态管理不同。react 中 flux、redux 等状态管理是社区去做的，vue 中 vuex 这种状态管理是自己来做的。

flux 与 redux

flux 中四大要素：view、action、dispatcher、store

redux 中六大要素：view、action、reducer、store、connect、provider。connect 的实现就是利用 provider 无层传递的 store，利用高阶组件的思想把 store、action 与 想要关联的组件传递进去返回一个新的组件。combineReducers 可以整和多个 reducers 为一个 reducer。

setState 为什么是异步的

  1. setState 同步时可能会引起不必要的渲染。
  2. setState 同步时无法保证 state 与 props 的统一性。
