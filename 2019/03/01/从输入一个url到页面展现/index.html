从输入一个url到页面展现

基础流程

  1. 在浏览器的地址输入 url 并按下回车。
  2. 浏览器查找当前 url 是否存在缓存，并检验缓存是否过期。
  3. DNS 解析 url 对应的 IP。
  4. 根据 IP 建立 TCP 连接。（三次握手）
  5. 客户端发起 HTTP、HTTPS 请求
  6. 服务端响应 HTTP、HTTPS 请求，客户端接收响应。
  7. 渲染页面，构建 dom 树。
  8. 断开 TCP 连接。（四次挥手）

使用缓存部分

强制缓存：判断上一次服务端返回 HTTP response Header 字段是否含有 Cache-Control:Max-Age=xxx（相对时间，max-age=0 不使用缓存） 或 Expires=xxx（绝对时间），有且没有过期，客户端使用本地缓存。

对比缓存：在没有强制缓存的情况下，客户端获取上一次服务端返回 HTTP response Header 中 E-Tag 或 Last-Modified 字段，再下一次 HTTP request Header 中添加 If-None-Match 或 If-Modified-Since 字段，服务端判断客户端发送的 HTTP request Header 中的 If-None-Match（客户端当前版本号） 或 If-Modified-Since （客户端版本最后更新时间）和 服务端中 E-Tag（服务端当前版本号） 或 Last-Modified（服务端版本最后更新时间）是否存在对应关系（If-None-Match === E-Tag 或 If-Modified-Since >= Last-Modified），存在对应关系则返回状态码 304，客户端使用本地缓存。

网络通信部分

DNS 解析：DNS 用于将域名解析成对应的 IP，首先客户端检测本地是否有域名对应 IP，有则使用 IP（不需要通过 DNS 进行解析），如果没有则去本地的 DNS 解析器中缓存寻找，如果还没有则通过本地的 DNS 服务器查询，依次查询顶级域名，二级域名，三级域名…对应的 DNS 服务，最后找到域名对应的 IP。

TCP 连接的建立：TCP 为可靠连接，提供字节流服务，UDP 为不可靠连接，面向报文。TCP 通过三次握手建立连接，过程中需要用到四种包（SYN 建立联机、ACK 确认、seq 顺序号码、ack 确认号码）。第一步客户端发送 SYN（SYN = 1，seq = i） 标志的数据包，请求服务器建立联机。第二步服务端返回 SYN/ACK （SYN = 1，ACK = 1，ack = i + 1，seq = j） 标志的数据包，确认收到联机信息，请求客户端建立联机。第三步客户端返回 ACK（ACK = 1，ack = j + 1） 标志的数据包，确认联机的建立。

HTTP、HTTPS 请求的传输：相对于 HTTP 协议，HTTPS 协议多了一层 TLS（SSL）加密协议，是在传输层中完成的，网络层管理结构：客户端应用层 -> 客户端传输层 -> 客户端网络层 -> 客户端数据链路层 -> 服务端数据链路层 -> 服务端网络层 -> 服务端传输层 -> 服务端应用层。在客户端传输过程中，首先应用层通过 HTTP 或 HTTPS 协议发出请求，请求包含请求行、请求头、请求主体几个部分。其次传输层端通过 TCP 协议传输报文，将大快数据分割成报文段形式，并为其编号，通过 TSL（SSL）协议为报文段加密。然后网络层通过 ARP 协议查询 MAC 地址，通过 IP 协议将 TCP 分割好的报文段传送给接收方。最后数据到达数据链路层，客户端部分传输结束。在服务端传输过程中，重复上述步骤由下至上，最终到达应用层，并作出响应再回传给客户端（html文档）。

TCP 连接的断开：TCP 通过四次挥手断开连接，过程中需要用到四种包（FIN 结束、ACK 确认、seq 顺序号码、ack 确认号码）。第一步开户端发送 FIN/ACK（FIN = 1，ACK = 1，seq = i） 标志的数据包，请求服务器结束联机。第二步服务端返回 ACK（ACK = 1，ack = i + 1）标志的数据包，确认收到结束信息。第三步服务端发送 FIN/ACK（FIN = 1，ACK = 1，seq = j）标志的数据包，请求客户端断结束联机。第四步客户端返回 ACK（ACK = 1，ack = j + 1），确认收到结束信息，并断开联机。第二步与第三步的过程中服务端会进入半关闭状态，并把没传递完成的数据继续传递完成。

页面渲染部分

DOM 树的构建：将 html 文档拆分成词（token），并存入栈中，根据状态机对词进行解析，生成 DOM 树。

CSSOM 树的构建：通过 CSS 匹配器找到对应的 DOM，并将 CSS 属性应用到 DOM 上，生成 CSSOM 树。

RENDER 树的构建：通过对 CSSOM 树上不同的规则（文字、盒、position、float、flex），进行不同的排版，生成 RENDER 树。

位图 与 合成层 的构建：在内存中建立 RENDER 树的位图，并对其中某些元素进行合成（排除一些可能变化的元素，例如：position、transform），建立合成位图，从而减少绘制的次数，提高浏览器性能。

页面的绘制：将最终显示的位图交给操作系统，最终把位图绘制到屏幕上。

附：在浏览器还没有接收到完整的 html 文件时，就开始渲染页面了，遇到外部链入的 脚本标签、样式标签、图片 时，再次发送 http 请求，重复上述步骤。收到 css 文件会对已经渲染的页面重新渲染，加入他们相应的样式，收到图片会立刻显示到相应位置。这一过程中可能会触发页面的重绘或重排，DOM 树每个节点都有重排的方法，由于他不仅影响自己，还可能影响其他的节点，所欲他耗费的性能要比重绘高的多，下述情况可能会发生重排：页面初始化、js 操作 dom、窗口 resize、css属性发生变化。
