网络攻击入门

发表于 2019-03-05 | 分类于 网络
网络攻击入门

XSS

xss：跨站点脚本攻击。

xss 的分类：

  1. 存储型 xss：攻击者将恶意代码提交到网站的数据库，用户打开该网页时，网站的服务器将恶意代码从数据库中取出，拼接在 html 中返回给客户端，用户浏览器收到代码后解释执行，并执行其中的恶意代码。（服务端 xss）
  2. 反射型 xss：攻击者构造恶意的 url，用户通过该 url 打开该网页时，网站的服务器将恶意代码从 url 中取出，拼接在 html 中返回给客户端，用户浏览器收到代码后解释执行，并执行其中的恶意代码。（服务端 xss）
  3. DOM 型 xss：攻击者通过构造恶意的 url 或 在网站进行恶意的输入时，网站的客户端根据 url 或 恶意输入，生成不正常的 dom 或为 dom 添加不正常的属性，从而执行恶意的代码。（客户端 xss，应用情况：document.write、innerHTML、location.href、iframe.src）

xss 的预防措施：

  1. 针对于前两种服务端的 xss 我们可以考虑使用纯前端渲染，通过 ajax 发请求获取数据，并对数据进行渲染。
  2. 针对于 DOM 型的 css 我们可以客户端部分的代码，对于能直接修改 dom 或是创建 dom 到 html 中的代码做好转义。（转义工具，例如：npm -> xss）
  3. 同时可以强化我们的 cookie 只在 http 请求中可用，设置响应头（Set-Cookie: HttpOnly;），恶意代码就算执行也窃取不到关键信息。

CSRF

csrf：跨站点伪造请求。

csrf 的攻击情形：登录受信用的网站 A，并在本地生 cookie，在不登出 A 的情况下，访问不受信用的网站 B，网站 B 通过伪造的请求进行恶意攻击。

csrf 的防御：

  1. 服务端通过在 cookie 中添加一个随机数并返回给客户端，客户端在每次请求中或在表单提交中添加这个随机数（为表单增加 hash值）传递给服务端，服务端校验通过方可生效。
  2. 如果不受信用的网站 B 存在 xss 攻击（不可控因素），则为了更安全，可以通过服务端返回客户端一个图形验证码或短信验证码，客户端手动输入该验证码和要传递信息一起传递给服务端，服务端校验通过方可生效。

DoS

dos：拒绝服务。

dos 的攻击情形：迫使服务器缓冲区满，不接受新的请求，导致用户无法正常访问该网站。

dos 的防御：采用流量过滤，拥塞网络带宽的流量将被过滤掉，正常的流量可以通过，可以使用防火墙、流量清洗等手段防范。

SQL 注入

sql 注入：执行错误的 sql 语句。

sql 注入的攻击情形：在访问的 url 上或在表单提交时构造不正确信息（sql 语句的一部分），从而使得服务端执行错误的 sql 语句。

sql 注入的防御：在客户端对提交的表单中的信息进行正则验证，在服务端对要执行的 sql 语句进行过滤，查看是否有非法字符。
