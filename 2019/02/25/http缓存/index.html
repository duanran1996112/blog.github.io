http缓存

前言

缓存的好处：
1.减少了冗余的数据传输，节省的网费。
2.减少了服务器的负担，提高了网站的性能。
3.加快了客户端加载网页的速度。

缓存

http 缓存是指，当 web 请求抵达缓存时，如果本地有已缓存的副本，就可以从本地设备，而不是原始的服务器中提取这个文档

强制缓存

对于强制缓存来说，响应的 header 中会有两个字段来表明是否失效（cache-control:max-age=xxx、expires），cache-control 的优先级高于 expires。

强制缓存 成功 & 失败

cache-control 的只可以有 private（默认）、public、max-age=xxx，no-cache，no-store。
private：客户端可以缓存
public：客户端与代理服务器均可缓存
max-age=xxx：缓存内容在 xxx 秒后失效，单位：秒（相对）
no-cache：强制缓存无效，使用对比缓存来验证
no-store：任何缓存均无效

expires 的值为服务器缓存的到期时间（时间戳），在到期时间内均可使用缓存数据（绝对）。

对比缓存

进行比较判断 请求（if-modified-since、if-none-match） 与 响应（last-modified、eTag） header 中的字段来决定是否可以使用缓存，若可以使用则返回状态码 304。

if-modified-since：上次请求时返回资源的最后修改时间（时间戳）
last-modified：服务器返回资源的最后修改时间（时间戳）
if-modified-since > last-modified ? 304（对比缓存） : 200（重新请求）

if-none-match：上次请求时服务器返回资源的唯一标识
eTag：服务器返回资源的唯一标识
if-none-match === eTag ? 304（对比缓存） : 200（重新请求）

if-none-match + eTag > if-modified-since + last-modified

强制开启缓存

应用场景：缓存静态文件（html、图片）
方法：
1.在服务器配置 cache-control:max-age 或 expires 为当前时间一年之后
2.在客户端设置 if-modified-since 的时间大于上次服务器修改的时间
