函数式编程

函数式编程定义

函数式编程：泛函编程，是一种编程范式，它将电脑运算视为数学上的函数计算。

个人理解：函数式编程是依赖纯函数的，同时函数式编程有两个最基本的运算：合成和柯里化。

纯函数（函数式编程的基石，无副作用的函数）

纯函数：
1.此函数在相同的输入值时，总是产生相同的输出。
2.此函数运行过程不影响运行环境。

javascript 内置纯函数：Array.prototype.slice、Array.prototype.map、String.prototype.toUpperCase 等…
javascript 内置非纯函数：Array.prototype.splice 等…

函数的合成

函数的合成：如果一个值要经过多个函数，才能变成另外一个值，就可以把所有中间步骤合并成一个函数，这叫做”函数的合成”。

注意：函数的合成，被合成的函数需要是一个纯函数。

前言：如果被合成的函数可以接受多个参数，函数的合成就会非常麻烦，所以就需要函数柯里化了。

函数的柯里化：就是把一个多参数的函数，转化为单参数函数。

函数式编程的一些高阶应用

Functor（函子）

函子：函子是函数式编程里面最重要的数据类型，也是基本的运算单位和功能单位。它的变形关系可以依次作用于每一个值，将当前容器变形成另一个容器。

函子代码的实现：任何具有 map 方法的数据结构都可以当做函子的实现。

一般约定，函子的标志就是容器具有 map 方法。该方法将容器里面的每一个值，映射到另一个容器。

Maybe（函子）

函子接受各种函数，处理容器内部的值。这里就有一个问题，容器内部的值可能是一个空值（比如 null ），而外部函数未必有处理空值的机制，如果传入空值，很可能就会出错。Maybe 函子为 map 方法设置了空值检查。

Either（函子）

条件运算 if else 是最常见的运算之一，函数式编程里面，使用 Either 函子表达。

Applicative（函子）

applicative 函子的意义在于，对于那些多参数的函数，就可以从多个容器之中取值，实现函子的链式操作。

Monad（函子）

Monad 函子的作用是，解决嵌套的函子，将嵌套的函子，总是返回一个单层的函子。它有一个 flatMap 方法，与 map 方法作用相同，唯一的区别是如果生成了一个嵌套函子，它会取出后者内部的值，保证返回的永远是一个单层的容器，不会出现嵌套的情况。

函数式编程与React的关系

1.React 的组件就是一个函数生成，React 中的高阶组件就是函数中的高阶函数，例如 Redux 中的 connect 和 react-router 就是放在 App 外的高阶函数。他们的原理就是将子组件传入父组件中，然后返回一个新的组件，可以复用父组件的逻辑。React 页面的渲染过程本质就是函数的嵌套调用过程。
2.React 中的子组件受父组件传入 props 的控制，但是子组件不可以直接修改 props，这有点类似于纯函数的特征，不改变外部状态。所以我们在开发组件时，将具有负作用的操作交给外部控制，这样组件才是相互独立的，具有极高的适应性。
3.在 Redux 中函数式编程的思想更为明显，Reducer 就是将一些固定的运算抽象出来，所以 Reducer 必须是一个纯函数，每次调用完成都返回一个新的对象，不直接修改状态。这样数据的操作更为清楚，不需要考虑运行的结果因其他的原因产生影响。
4.React 和 Redux 当中也使用了很多柯里化和函数合成的写法。

函数式编程的好处

1.开发效率快，高度使用函数可以不断服用逻辑，同时函数对外无副作用。
2.通过函数名称可以直接理解其作用，接近自然语言。
3.代码清晰，真正调用的代码简单，所有细节逻辑都封装在函数内。
4.方便处理并发，因为纯函数不影响外部变量。
